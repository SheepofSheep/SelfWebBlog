# 更新日志

## 2026-06-01 - PRD P0 上线安全优化

### 后端

- 修复草稿文章可被直接访问的问题：`GET /posts/{id}` 现在会拦截非管理员读取 `DRAFT` 状态文章。
- 修复评论可以挂到不存在文章或草稿文章的问题：发表评论前会校验目标文章存在且已发布。
- 优化文章更新逻辑：编辑已有文章时先读取旧数据再合并，避免前端未传字段时清空摘要、封面、分类、标签等已有内容。
- 增加文章状态校验：只允许 `DRAFT` 和 `PUBLISHED`，非法状态会被拒绝。
- 将草稿、搜索、分类查询方法上移到 `PostService` 接口，移除 Controller 对实现类的强转依赖。
- 新增控制器单元测试，覆盖草稿访问、管理员草稿访问、文章安全更新、非法状态、评论目标文章校验。

### 前端

- 优化 API 错误处理：当接口返回未登录、无权限、401/403 或 token 相关错误时，自动清理本地登录态。
- 增加全局登录失效事件处理：受保护页面遇到登录失效会提示并跳转登录页。

### 工程与安全

- 更新 `.gitignore`，统一忽略依赖目录、构建产物、本地环境变量、SQLite 数据库及 WAL 文件、上传目录、本机凭据和临时文件。
- 保留 `.env.example` 作为可提交模板，避免真实环境变量进入 Git。

### 验证

- `mvn test` 通过：7 个测试，0 失败。
- `pnpm build` 通过：前端生产构建成功。

### 上线注意

- 上线前仍需先备份服务器的 `blog.db` 和 `/uploads`。
- 本次没有变更数据库结构，不需要执行迁移脚本。
- 部署后建议验证：
  - 未登录访问草稿详情应返回不可见。
  - 管理员草稿箱仍可正常编辑草稿。
  - 登录用户不能对草稿或不存在文章发表评论。
  - 管理页登录过期后能正确跳转登录页。
